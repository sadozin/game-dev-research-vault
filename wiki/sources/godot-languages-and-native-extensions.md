# Source: Godot C#, GDScript, and GDExtension documentation

- **Author:** Godot Engine contributors
- **Published:** continuously maintained documentation
- **URL:** https://docs.godotengine.org/en/stable/engine_details/engine_api/gdextension/index.html
- **Retrieved:** 2026-07-11

## Summary

Godot supports GDScript for tightly integrated iteration, C# for .NET-oriented workflows, and
GDExtension for native libraries that interact with the engine API without rebuilding the engine.
Language choice changes tooling, deployment, interop, and hot-path costs; native extension code also
inherits ABI, memory, and thread-safety responsibilities.
